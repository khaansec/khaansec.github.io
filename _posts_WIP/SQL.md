I recently started getting back into SQL. It has probably been 4 years or so since I looked into it last. I am still not super far into that journey yet, but wanted to take not of some new things I have learned and some changes since the last time I looked at it.

Glad to see that SSMS (SQL Server Management Studio) is still a thing, as that is what I learned on. That looks mostly the same  and it is as easy as ever to import something like AdventureWorks. 

Of course, the red-teaming thoughts in the back of my mind make me think about how to interact with things in ways beyond what is presented, so I did a bit of digging. As far as I can tell, sqlcmd is installed along with SSMS, which means that any authenticated PS Session or PSexec session can easily interact with the server. Sqlcmd can be used by entering calling it as a command, entering a query like "SELECT * FROM sys.databases;", then typing "go". Alternatively, you can pass a query
